# Тренажёр 3.1: Знакомство с диалоговым проектом

1. Склонируйте репозиторий gd-trainer-02-dialogue
   через GitHub Desktop или Git:
   • Откройте ссылку на репозиторий из материалов курса.
   • Нажмите кнопку Code → выберите Open with GitHub Desktop или скопируйте URL для git clone .
   • Укажите папку на диске для проектов курса.
   • Дождитесь окончания клонирования.
2. Добавьте проект в Unity Hub:
   • Откройте Unity Hub → вкладка Projects.
   • Нажмите Add → Add project from disk.
   • Выберите папку gd-trainer-02-dialogue на диске.
   • Нажмите Select Folder.
3. Откройте проект в нужной версии Unity:
   • В списке проектов найдите gd-trainer-02-dialogue.
   • Убедитесь, что выбрана версия Unity 6.3 LTS.
   • Откройте проект и дождитесь импорта.
   • Проверьте вкладку Console.
4. Откройте сцену DemoScene:
   • В окне Project разверните Assets/_Project/Scenes .
   • Дважды кликните DemoScene.unity .
   • Убедитесь, что в Hierarchy отображаются основные объекты (Canvas, NPC, менеджеры).
5. Запустите диалоговую сцену:
   • Нажмите Play.
   • Переключитесь на вкладку Game.
   • Пройдите диалог до конца, прочитав все реплики.
   • Обратите внимание, как меняются варианты ответов и реплики.
6. Найдите ScriptableObjects диалогов:
   • В окне Project разверните Assets/_Project/Dialogues .
   • Просмотрите несколько DialogueNode(двойной клик по asset).
   • Найдите поля speakerName, dialogueText, choices.
7. Найдите Canvas диалога:
   • В Hierarchy найдите объект DialogueCanvas.
   • Выделите его и посмотрите структуру UI: DialoguePanel, SpeakerName, DialogueText, контейнер кнопок.
   • Убедитесь, что элементы совпадают с тем, что вы видели в Game View.

# Тренажёр 3.2: Настройка UI-элементов диалога

1. Откройте сцену DemoScene
   и найдите объект DialogueCanvas
   в Hierarchy. Выделите элементы DialoguePanel, SpeakerName, DialogueText
   и шаблон ChoiceButton, чтобы увидеть их параметры в Inspector.
2. Настройте фон панели диалога DialoguePanel:
   • Увеличьте непрозрачность (Alpha) до 0.9, чтобы текст уверенно читался на фоне сцены (ранее могло быть 0.5).
   • При необходимости слегка затемните цвет панели.
3. Обновите оформление имени говорящего SpeakerName:
   • Установите размер шрифта около 24.
   • Сделайте имя визуально чуть более заметным (например, полужирным).
   • При необходимости измените цвет шрифта, чтобы он хорошо читался на фоне панели.
4. Настройте текст реплики DialogueText:
   • Установите размер шрифта 18–20 (не оставляйте значения уровня 12–14).
   • Проверьте читаемость на обычном мониторе и ноутбуке.
   • При необходимости поправьте межстрочный интервал.
5. Настройте внутренние отступы (Padding) у панели с текстом DialogueText
   или контейнера, в котором он расположен:
   • Поставьте отступы порядка 15–20 пикселей со всех сторон.
   • Убедитесь, что длинные строки не прилипают к краю.
6. Обновите базовый цвет кнопки выбора ChoiceButton (Normal Color):
   • Выберите спокойный базовый цвет (например, светло-синий #5B9BD5 или аналогичный по тону).
   • Убедитесь, что цвет контрастирует с текстом на кнопке.
7. Настройте подсветку при наведении и нажатии у ChoiceButton:
   • Для Highlighted Color выберите цвет чуть ярче базового (например, #2E75B6).
   • Для Pressed Color выберите более тёмный вариант (например, #1F4E79).
   • Проверьте поведение в Play Mode.
8. Проверьте текст на кнопках ChoiceButton:
   • Установите размер шрифта около 16–18 pt.
   • Убедитесь, что текст выровнен по центру и не обрезается при длинных подписях.
   • При необходимости расширьте область кнопки или настроьте перенос строк.
9. Сравните ваш результат с референсом Docs/UI_Reference.png
   и при необходимости подправьте отступы, размеры и цвета.

# Дополнительный модуль: Экранные переходы (Flow) на основе готового проекта

1. Откройте сцену с UI и найдите панели MainMenu, Settings, GameScreen, PausePanel, а также объект с компонентом UIManager .
2. Настройте кнопку MainMenu / Играть:
   • В Inspector добавьте событие OnClick.
   • Перетащите объект с UIManager .
   • Назначьте вызов метода ShowPanel("GameScreen")и при необходимости HidePanel("MainMenu").
3. Настройте кнопку MainMenu / Настройки:
   • Добавьте OnClick с вызовом ShowPanel("Settings").
   • Спрячьте главное меню через HidePanel("MainMenu").
4. Настройте кнопку Settings / Назад:
   • Добавьте OnClick с вызовом ShowPanel("MainMenu") и HidePanel("Settings").
5. Настройте кнопку паузы на игровом экране GameScreen / Пауза:
   • Добавьте OnClick с вызовом ShowPanel("PausePanel").
   • В GameManager или другом подходящем компоненте обеспечьте остановку времени (например, Time.timeScale = 0f;).
6. Настройте кнопку PausePanel / Продолжить:
   • Добавьте OnClick с вызовом HidePanel("PausePanel").
   • Верните время в норму (например, Time.timeScale = 1f;).
7. Настройте кнопку PausePanel / В меню:
   • Добавьте OnClick с вызовом ShowPanel("MainMenu")и HidePanel("PausePanel").
   • При необходимости спрячьте GameScreen , чтобы не оставаться в игровой сцене.
8. Протестируйте все переходы по кругу: вход из главного меню в игру, переход в настройки и обратно, постановка игры на паузу и возврат в меню.

# Тренажёр 3.3: Ветвящийся диалог без условий

1. Создайте стартовый DialogueNode для торговца:
   • ПКМ в Assets/_Project/Dialogues → Create → Dialogue → Node.
   • Назовите asset, например, Merchant_Start .
   • Заполните speakerName = "Торговец".
   • В dialogueText напишите приветствие: «Добро пожаловать, путник! Чем могу помочь?».
2. Добавьте три варианта ответа (choices):
   • "Покажи товары".
   • "Расскажи о себе".
   • "Прощай".
   • Для каждого создайте пустую ссылку nextNode(пока null).
3. Создайте ветку «Покажи товары»:
   • Создайте новый DialogueNode , например Merchant_ShowGoods .
   • speakerName = "Торговец", dialogueText с коротким описанием ассортимента.
   • В choices добавьте вариант "Вернуться", ведущий обратно на Merchant_Start.
4. Создайте ветку «Расскажи о себе» с двумя уровнями:
   • Узел Merchant_About_1 с первой частью истории.
   • В choices добавьте: - "Интересно, продолжай"→ Merchant_About_2 ."Хватит болтать" → возврат на Merchant_Start .
   • Узел Merchant_About_2 с продолжением истории и одним вариантом ответа "Вернуться"→ Merchant_Start.
5. Создайте ветку завершения «Прощай»:
   • Узел Merchant_Bye с короткой репликой прощания.
   • В choices либо не добавляйте вариантов (диалог завершается автоматически), либо добавьте один вариант "Завершить диалог", который сообщает менеджеру диалогов о конце.
6. Привяжите стартовый узел к NPC:
   • Выберите объект или префаб NPC_Merchant .
   • В компоненте диалоговой системы найдите поле startDialogue .
   • Установите туда Merchant_Start.
7. Протестируйте все ветки:
   • Запустите сцену и поочерёдно пройдите через все варианты.
   • Убедитесь, что нет тупиков и зацикливаний, кроме осознанных возвратов к началу.

# Тренажёр 3.4: Условные ветки от состояния игрока

1. Сделайте приветствие торговца зависимым от репутации игрока:
   • Создайте два стартовых узла: Merchant_Start_Neutral и Merchant_Start_Hostile .
   • В Merchant_Start_Neutral задайте приветливый текст и настройте showCondition с типом HasReputation для неотрицательной репутации.
   • В Merchant_Start_Hostile задайте более холодный/грубый текст и настройте showCondition для отрицательной репутации.
   • Убедитесь, что DialogueManager выбирает подходящий узел в зависимости от PlayerState.reputation.
2. Добавьте условный вариант скидки при высокой репутации:
   • В ветке диалога (например, «Покажи товары») добавьте вариант ответа "Скидка для друзей".
   • В showCondition этого выбора установите тип HasReputation и пороговое значение (например, requiredValue = 30).
   • Протестируйте поведение диалога при низкой и высокой репутации.
3. Сделайте отдельную ветку диалога, зависящую от квестового флага:
   • Добавьте в PlayerState.flags флаг, например "quest_01_complete".
   • Создайте узел благодарности торговца за выполненный квест.
   • Настройте у этого узла showCondition с типом HasFlag и именем флага "quest_01_complete".
   • Проверьте появление этой ветки до и после установки флага (через Debug‑панель или геймплей).
4. Настройте покупку предмета, зависящую от количества золота:
   • Добавьте вариант выбора "Купить зелье (100 монет)".
   • В его showCondition установите тип HasGold и requiredValue = 100 .
   • В обработчике выбора уменьшите значение gold на 100 и выведите реплику об успешной покупке.
   • Протестируйте покупку при gold < 100 и gold >= 100.

# Тренажёр 3.5: Мини‑QA диалогов и полировка

1. Откройте файл чек-листа Docs/QA_Checklist.md
   и внимательно изучите все пункты, на которые вам нужно обратить внимание при тестировании.
2. Запустите проект и по очереди пройдите все диалоги, параллельно отмечая пункты чек-листа:
   • Проверяйте достижимость всех веток.
   • Отмечайте, из каких состояний нельзя выйти.
   • Следите за текстом, портретами и условиями.
3. Найдите минимум 3 бага в диалогах или интерфейсе:
   • Неверные или отсутствующие портреты.
   • Тупиковые ветки без выхода.
   • Условия, которые срабатывают неправильно.
   • Обрезанный, налезающий или нечитаемый текст.
4. Для каждого найденного бага заполните шаблон Docs/BugReport_Template.md:
   • Придумайте понятный заголовок.
   • Опишите проблему и шаги для воспроизведения.
   • Зафиксируйте ожидаемое и фактическое поведение.
   • При возможности приложите скриншоты.
5. Исправьте найденные баги в проекте:
   • Обновите нужные DialogueNode , портреты, условия Condition или настройки UI.
   • Проверьте, что правки не ломают другие ветки и экраны.
6. Повторно пройдите по чек-листу Docs/QA_Checklist.md , чтобы убедиться, что баги не воспроизводятся и не появилось новых критичных проблем.
7. Запишите 2–3 идеи по улучшению UX диалогов (в отдельном файле или в конце чек-листа):
   • Например, добавить звук выбора, изменить скорость появления текста, улучшить подсветку текущей реплики или выбора.
