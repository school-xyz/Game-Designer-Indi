### Тренажёр 3.1: Знакомство с диалоговым проектом
1. [ ] Проект открывается и запускается без ошибок.
2. [ ] Диалог полностью проигрывается.
3. [ ] Студент может показать, где лежат ScriptableObjects диалогов и DialogueCanvas.

### Тренажёр 3.2: Настройка UI-элементов диалога

1. [ ] Текст диалога и имена персонажей легко читаются.
2. [ ] Кнопки выбора имеют понятный визуальный отклик.
3. [ ] Итоговый UI в целом похож на референс и не «сыпется» при длинных строках текста.

### Дополнительный модуль: Экранные переходы (Flow) на основе готового проекта

1. [ ] Все кнопки ведут на ожидаемые экраны.
2. [ ] Нет «мёртвых» экранов, из которых нельзя вернуться.
3. [ ] Пауза действительно останавливает и возобновляет игру.

### Тренажёр 3.3: Ветвящийся диалог без условий

1. [ ] Все три варианта ответа из стартового узла работают.
2. [ ] Ветка «о себе» имеет минимум два уровня глубины.
3. [ ] Из любой ветки можно вернуться или завершить диалог.
4. [ ] Нет неожиданных тупиков и бесконечных зацикливаний.

### Тренажёр 3.4: Условные ветки от состояния игрока

1. [ ] Диалог меняется при разных значениях reputation.
2. [ ] Условные варианты появляются/пропадают корректно.
3. [ ] Флаги и золото реально влияют на доступность веток.

### Тренажёр 3.5: Мини‑QA диалогов и полировка

1. [ ] Найдены и задокументированы минимум 3 бага.
2. [ ] Баги исправлены и не воспроизводятся повторно.
3. [ ] Студент может сформулировать конкретные идеи по улучшению UX.
